# Design Context — starkblaze01.dev (v2 — "Trainer Card")

## Product

Personal portfolio for **Mayank Pathela** (*starkblaze01*).
Deployed at <https://starkblaze01.dev/> on Netlify.
Stands as a public résumé and personal log of a career since 2019.

## Audience

- Hiring managers, engineering leads, and founders evaluating Mayank for
  senior full-stack / platform roles (Dubai, remote, Atlanta via Georgia Tech).
- Developer peers from GitHub, dev.to, Twitter, open-source mentoring circles.
- Himself — permanent record of what he's shipped, where he's been.

## Concept

**"Trainer Card."** The site uses Pokémon as its *structural metaphor*,
never as decoration.

- **Content is serious.** Work history, writing, projects, mentoring — all
  written as a professional would write them.
- **Frame is playful.** The Home page opens as a stylized trainer card.
  Credentials (GSoC mentor, Outreachy mentor, Microsoft Student Ambassador,
  IIT Bombay research intern, ICDCIT publication, Georgia Tech MSCS) are
  rendered as *badges earned* — a Pokémon-trainer analogue for an academic
  CV bullet list.
- **Projects use type labels** (Electric/Psychic/Steel/Fighting/Normal) as
  a tiny delight layer.
- **One easter egg** (Konami code) reveals a hidden Pokédex overlay.

**If a recruiter reads only the prose, they see a professional.** The
trainer-card frame reveals character to whoever looks closer.

## Palette (single-accent with a story)

- **Ink** `#0a0a0f` — background.
- **Pokéball red** `#e63946` — single accent. Reserved for one emphasis
  per view. Used on the trainer-card corner, the primary CTA, the active
  nav link. Never on body text.
- **Cream** `#f2ead3` — for "card stock" surfaces (trainer card, badge
  chips), display-font headings on dark.
- **Off-white** `#e9e7e1` — body text on ink. Never pure white.
- **Muted type colors** — used only on project badges, desaturated versions
  of the canonical Pokémon type palette (never the saturated kid-show
  versions). Electric is ochre, not neon yellow; Psychic is dusty rose,
  not hot pink.

## Typography

- **Display / headings**: **Fraunces** (variable; dial SOFT up slightly for
  warmth). Serif, distinctive, not overused. All H1s and H2s.
- **Body**: **Inter** (variable). Clean and legible. Body prose only — not
  headings; that's the AI-slop cliché we avoid.
- **Mono / labels / badge IDs**: **JetBrains Mono**. Trainer-card IDs,
  badge captions, small labels.

Max line length on prose: ~65ch. Body line-height: 1.65. Dark-on-ink
type gets +0.05 line-height (thin strokes need room).

## Motion

- Minimal. One signature animation: a **typed cursor effect** on "Hi There!"
  at the top of About.
- Card hover: 120 ms translate + soft shadow. No bounces, no spring.
- `prefers-reduced-motion` honored — animations shrink to opacity fades.

## What's OUT

- Particle backgrounds (v1 identity, retired with the refresh).
- Light/dark switch (one confident palette beats a weak toggle).
- Gradient text, glassmorphism, hero metrics, "Trusted by" rows.
- Emoji spam, sprite graphics, pixel fonts (the Pokémon layer is a
  *metaphor*, not a retro-game cosplay).
- Coral (v1 accent) and Roboto Slab (v1 body) — both retired.

## Constraints

- **Stack**: Vite 6, React 18, TS 5 strict, Redux Toolkit, Tailwind 3,
  AntD 5 (used sparingly — Spin mostly), React Router 6. Native fetch.
- **Performance**: first-paint JS ≤ 200 KB gzip; non-critical routes lazy.
- **A11y**: WCAG AA. Contrast ≥ 4.5:1 on red-on-ink and cream-on-ink.
  Every interactive element keyboard-reachable with a visible focus ring.
- **SEO**: per-route `<title>` + `<meta description>`, `Person` JSON-LD
  on Home, `robots.txt` + `sitemap.xml`, semantic heading hierarchy,
  static prerender where feasible.

## Differentiation — the unforgettable

A developer portfolio where **your mentoring history and academic
credentials are rendered as Pokémon badges**. Nothing else on the web
looks like it, and it still reads as professional from the first scroll.
